## 重要な設計方針

### ブラウザ UI のパフォーマンス
- ブラウザ DOM には**直近 25 行のみ**マウントしてください
- 古い行は DOM から削除（`container.removeChild(container.firstChild)`）
- 全テキストは Python 側の `session_store.py` がメモリ + JSON で保持します
- 要約は DOM からテキストを取得するのではなく、サーバー側の `session_store.get_full_text()` から取得してください
- 全文を見返す機能が必要な場合は `/api/session/transcript` エンドポイントを用意し、ページネーション付きで返してください

### 翻訳
- Gemini 3.1 Flash-Lite を使用（`google-genai` パッケージ）
- 短いテキストはバッチで溜めてからまとめて翻訳し、API 呼び出し回数を削減してください
- API エラー時は原文をそのまま表示するフォールバックを必ず入れてください

### 要約
- Gemini 3 Flash を使用
- 概要 / 詳細 / アクションアイテム の3種類のテンプレートを実装してください
- 長時間セッション（コンテキスト超過）の場合はチャンク分割 → 個別要約 → 統合要約の流れで処理してください

### 話者分離
- `config.enable_diarization = False` をデフォルトにして、明示的に有効化する設計にしてください
- HuggingFace トークンが未設定の場合はエラーではなく警告を出してスキップしてください

### 環境変数
- `.env` ファイルから `GEMINI_API_KEY` と `HF_TOKEN` を読み込んでください（`python-dotenv` を使用）
- API キーが未設定の場合、該当プラグインを無効化して起動できるようにしてください

## 既存コードとの統合
Phase 1〜3 で実装済みの以下のファイルに変更が必要です:
- `config.py` — 翻訳・要約・話者分離の設定項目を追加
- `main.py` — プラグインの初期化と on_text コールバックの拡張
- `server.py` — 要約・セッション管理の API エンドポイント追加
- `static/index.html` — 翻訳表示、話者ラベル、要約パネル、25行制限の追加
- `requirements.txt` — 依存パッケージ追加
